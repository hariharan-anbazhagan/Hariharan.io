Role & Output Expectation

Act as an expert frontend architect and portfolio website builder.
You must design and generate a simple, clean, responsive portfolio website that serves as an alternative to my resume.
The output should be production-ready, device-responsive, and pixel-aligned, following modern UI/UX standards.

Website Purpose

This website replaces my traditional resume.

It must clearly communicate my profile, experience, skills, projects, and contact information.

The site must work flawlessly on mobile, tablet, laptop, and desktop devices.

Mandatory Sections

The website must include exactly the following sections:

Hero (Home) Section

Experience Section

Skills Section

Projects Section

Contact Section

🔹 Section-by-Section Requirements
1. Hero (Home) Section

Layout must be two-column:

Left side: Profile image

Right side: Resume objective text

Profile image requirements:

Use my real image

Remove background only for my image

Add a yellow circular background layer behind the image

Image must scale properly and stay centered on all screen sizes

No distortion or cropping

Objective text:

Clear, professional summary

Resume-style objective

Vertically aligned with the image

2. Experience Section

Provide a brief but clear explanation of my career experience

Include:

Roles handled

Key responsibilities

Activities I managed

Content must be:

Easy to scan

Professionally worded

Suitable for recruiters

3. Skills Section

Use card-based UI

Create two main cards only:

Pharma Skills

IT Skills

Interaction behavior:

When a card is clicked:

It should expand or zoom

Text should become larger and easier to read

Display detailed skill information

Same behavior must apply to both cards

Must be keyboard and mobile friendly

4. Projects Section

Display projects using cards

Each project card must include:

Project title

Short description

GitHub repository link (clickable)

Cards must be:

Uniform in size

Responsive

Visually clean

5. Contact Section

Must include:

A relevant image or illustration

A contact form with fields:

Name

Email

Contact Number

Submit behavior:

On form submission:

An email must be triggered to the user

Email should include:

My contact details

A default professional quote/message

UI must clearly confirm successful submission

🔹 Technical & Design Constraints

Fully responsive layout

Pixel-perfect alignment

Clean typography

Minimal but modern design

No unnecessary animations

Accessibility-friendly (readable fonts, contrast)

Code must be structured and maintainable

Final Instruction

Do not add extra sections.
Do not change the section order.
Do not assume missing content—use placeholders where required.
Prioritize clarity, responsiveness, and professional presentation.
